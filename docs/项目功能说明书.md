# 水利信息管理系统项目功能说明书

## 1. 文档说明

本文档用于说明水利信息管理系统的业务功能组成、各模块页面用途、用户操作方式、页面流程和模块之间的业务关系，作为系统实现、演示和答辩时的功能依据。本文档不展开非功能需求、接口规范和技术结构，相关内容分别详见《项目规格需求说明书》和《项目结构说明书》。

## 2. 项目功能概述

本系统面向水利信息管理场景，提供工程管理、站点管理、监测数据、告警处理、图表统计和地图展示等用户可直接使用的业务功能，项目背景与建设目标详见《项目规格需求说明书》。

## 3. 用户与角色功能

### 3.1 用户角色

系统采用两类基础角色：

- 管理员
- 普通用户

### 3.2 角色功能范围

管理员在系统中的主要使用范围包括：

- 登录系统
- 查看和维护各类基础信息
- 管理站点和监测数据
- 查看和处理告警
- 查看图表和地图信息

普通用户在系统中的主要使用范围包括：

- 登录系统
- 查看基础信息
- 查看监测数据和统计结果
- 查看地图和告警信息

## 4. 功能模块说明

### 4.1 用户与权限管理模块

#### 模块作用

用于完成用户登录、身份识别和角色化访问控制，是所有业务页面的统一入口。

#### 用户使用方式

- 用户在登录页输入用户名和密码完成身份校验
- 登录成功后进入系统首页并加载个人角色对应的菜单
- 用户可在系统内查看当前登录信息并退出系统

#### 页面表现

- 登录页提供账号密码输入框和登录按钮
- 顶部或用户中心页面显示当前用户信息
- 不同角色进入系统后看到的操作范围不同

### 4.2 水利工程信息管理模块

#### 模块作用

用于维护水库、河道等基础工程信息，为站点绑定、地图展示和后续统计提供基础资料。

#### 用户使用方式

- 用户在列表页查看已有工程信息
- 用户可按条件查询目标工程
- 管理员可新增、编辑和删除工程资料
- 用户可进入详情页查看工程的完整字段信息

#### 页面表现

- 列表页展示名称、位置、管理单位等核心字段
- 表单页支持录入容量、流域、描述等信息
- 详情页用于集中查看单条工程记录

### 4.3 监测站点管理模块

#### 模块作用

用于维护监测站点基础资料，支撑监测数据记录、告警判断和地图点位展示。

#### 用户使用方式

- 用户在站点列表页查看各类站点信息
- 用户按站点类型、状态等条件筛选目标站点
- 管理员可维护站点经纬度、类型和运行状态
- 用户可通过详情信息了解站点位置和业务属性

#### 页面表现

- 列表页显示站点名称、类型、状态和坐标信息
- 表单页支持新增和修改站点资料
- 状态标签用于突出正常、离线和异常状态

### 4.4 监测数据管理模块

#### 模块作用

用于管理站点上报或人工录入的监测数据，并支持历史记录查询与趋势查看。

#### 用户使用方式

- 用户录入水位、雨量、流量等监测数据
- 用户按站点和时间范围筛选历史记录
- 用户在页面中查看表格结果和趋势变化

#### 页面表现

- 查询区域提供站点、时间范围等筛选条件
- 列表区域展示监测值、数据类型和采集时间
- 图表区域辅助查看历史趋势变化

### 4.5 告警管理模块

#### 模块作用

用于展示异常监测结果生成的告警记录，并支持用户查看、跟踪和处理告警状态。

#### 用户使用方式

- 用户查看告警列表，快速了解当前异常情况
- 用户通过等级、状态等条件筛选告警信息
- 管理员对未处理告警进行状态更新
- 用户进入详情页查看告警来源和告警说明

#### 页面表现

- 列表页突出告警等级和处理状态
- 详情页展示告警时间、站点来源和告警内容
- 状态更新操作形成处理闭环

### 4.6 数据统计分析模块

#### 模块作用

用于将业务数据转化为图表结果，帮助用户更直观地理解监测变化、站点分布和告警情况。

#### 用户使用方式

- 用户在首页或分析页面查看统计图表
- 用户通过图表快速掌握近期趋势和异常情况
- 用户结合列表和图表完成数据比对

#### 页面表现

- 首页展示核心统计卡片和趋势图
- 图表区域可展示水位、雨量、告警和站点状态结果
- 页面布局以直观展示和答辩演示为主

### 4.7 地图展示模块

#### 模块作用

用于在地图上展示水利工程和监测站点的空间位置，帮助用户从地理视角查看系统数据。

#### 用户使用方式

- 用户进入地图页查看工程和站点分布
- 用户点击不同点位查看对象详情
- 用户结合地图和业务信息理解空间分布关系

#### 页面表现

- 地图支持点位加载和分类展示
- 弹窗展示对象名称、类型和关键说明
- 页面重点突出空间分布和交互查看效果

## 6. 功能流程说明

### 6.1 登录功能流程

1. 用户输入用户名和密码
2. 前端提交登录请求
3. 后端校验账户信息
4. 返回 Token 与用户资料
5. 前端保存登录状态并进入系统

### 6.2 监测告警流程

1. 系统录入或接收监测数据
2. 后端根据阈值进行判断
3. 发现异常则生成告警记录
4. 用户在告警页面查看并处理告警

### 6.3 地图展示流程

1. 前端加载地图组件
2. 请求工程与站点数据
3. 在地图上渲染点位
4. 用户点击点位查看详情弹窗

## 7. 模块关系说明

系统主要模块关系如下：

- 用户与权限模块为所有业务模块提供访问控制
- 工程管理模块为地图展示提供工程基础数据
- 站点管理模块为监测数据和告警管理提供站点基础数据
- 监测数据模块为统计分析和告警管理提供数据来源
- 告警管理模块基于监测数据结果生成业务反馈
- 地图与图表模块为业务数据提供可视化展示

## 8. 功能亮点

- 功能模块完整，覆盖课程项目常见业务链路
- 同时具备管理、监测、告警、统计和地图展示能力
- 页面结构清晰，便于系统演示和答辩说明
- 功能设计扩展性较强，后续可继续增加业务模块
